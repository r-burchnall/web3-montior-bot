# FCSC Monitoring Architecture

## Overview

Monitoring system for the FastCarSlowCar MEV arbitrage bot fleet. Provides at-a-glance connectivity health, Yellowstone Geyser subscription status, and centralized searchable logs across all developer VMs.

## Components

### fcsc-agent (Go, per VM)

A single static binary (~11MB) that runs on each developer VM as a systemd service.

**What it does:**
- Tails the most recent `metrics.log*` file and parses QUEUE statistics
- Extracts Geyser subscription health from the bot's own log output
- Runs periodic connectivity checks against ClickHouse and Solana RPC
- Detects if the bot process is running
- Discovers the git branch of the traders-garage repo
- Exposes all data as Prometheus metrics on `:9100/metrics`

**What it does NOT do:**
- Connect to the Geyser gRPC stream directly (the bot already does this)
- Run any queries against the RPC beyond a simple reachability check
- Add meaningful load to Fastcar 1

**Configuration:** CLI flags baked into the systemd unit file. Generated by the interactive wizard (`scripts/configure-agent.sh`).

```
fcsc-agent \
  --listen-addr :9100 \
  --rpc-url http://64.130.40.37:8899 \
  --clickhouse-url http://monitoring.lxd:8123 \
  --log-scan-paths /home \
  --enable-checks all
```

### Promtail (per VM)

Ships bot log files to Loki on monitoring.lxd.

- Discovers files matching `/home/*/traders-garage/logs/metrics.log*`
- Extracts timestamp, log level, and thread ID via regex pipeline
- Labels each line with `vm` (hostname) and `level`
- 7-day retention enforced by Loki

### Monitoring Stack (monitoring.lxd)

Docker Compose stack managed by this repository:

| Service | Purpose | Port |
|---------|---------|------|
| Prometheus | Scrapes fcsc-agent metrics from all VMs | 9090 |
| Loki | Receives and stores logs from Promtail | 3100 |
| Grafana | Dashboards and alerting | 13000 |
| Tempo | Distributed tracing (existing) | 3200 |
| ClickHouse | Metrics storage (existing) | 8123 |
| cAdvisor | Container metrics (existing) | - |

## Metrics

All metrics are prefixed with `fcsc_` and labeled with `vm` (hostname).

### Bot Status
| Metric | Type | Description |
|--------|------|-------------|
| `fcsc_bot_running` | Gauge | 1 if bot process is running, 0 if not |
| `fcsc_bot_info` | Gauge | Always 1, carries `git_branch` label |

### Geyser Health
| Metric | Type | Description |
|--------|------|-------------|
| `fcsc_geyser_recv_rate` | Gauge | Geyser subscribe receive rate (msg/s) |
| `fcsc_geyser_send_rate` | Gauge | Geyser subscribe send rate (msg/s) |
| `fcsc_geyser_backlog` | Gauge | Messages queued but not yet processed |
| `fcsc_geyser_total_in` | Gauge | Cumulative messages received |
| `fcsc_geyser_total_out` | Gauge | Cumulative messages sent downstream |

### Queue Metrics (per queue)
| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `fcsc_queue_send_rate` | Gauge | `queue` | Send rate for quoter/searcher/executor/storage/pool_loader |
| `fcsc_queue_recv_rate` | Gauge | `queue` | Receive rate |
| `fcsc_queue_backlog` | Gauge | `queue` | Pending messages |

### Connectivity
| Metric | Type | Description |
|--------|------|-------------|
| `fcsc_clickhouse_up` | Gauge | 1 if ClickHouse responds to SELECT 1 |
| `fcsc_rpc_up` | Gauge | 1 if Solana RPC responds to HTTP |
| `fcsc_log_last_line_timestamp` | Gauge | Unix timestamp of last parsed log line |

## Health State Logic

Health states are computed in Grafana, not in the agent. The agent only exposes raw values.

| State | Condition | Meaning |
|-------|-----------|---------|
| **Green** | `fcsc_geyser_recv_rate > 0` for 3+ consecutive scrapes | Receiving Geyser updates normally |
| **Yellow** | `fcsc_geyser_recv_rate == 0` for 1-2 consecutive scrapes | Possible transient issue |
| **Red** | `fcsc_geyser_recv_rate == 0` for 3+ consecutive scrapes OR `fcsc_bot_running == 0` | Geyser feed lost or bot is down |

## Grafana Dashboards

### Fleet Overview
Grid view of all VMs. At a glance: Geyser health (color-coded stat panels), bot status (UP/DOWN), connectivity matrix (table with color-coded columns for Bot, Geyser, ClickHouse, RPC), time series of geyser rates and backlogs across the fleet, and log freshness.

### VM Detail
Deep dive into a single VM selected via dropdown. Status bar with 6 indicators, time series for geyser send/recv/backlog/totals, all queue metrics (send rate, recv rate, backlog), and a live log stream from Loki.

### Log Explorer
Designed for non-technical users. Dropdowns for VM (multi-select), log level, and free-text search. Shows log volume over time, error/warning rate chart, and a main log stream with combined filters. Expandable "Quick Searches" section with pre-built queries for arb executions, geyser events, errors, and pool loading.

## Deployment

### Build
```bash
./scripts/build.sh    # Produces build/fcsc-agent (linux/amd64)
```

### Configure a VM
```bash
./scripts/configure-agent.sh     # Interactive wizard -> systemd unit + YAML
./scripts/configure-promtail.sh  # Interactive wizard -> promtail config
```

### Deploy to VMs
```bash
# Single VM via LXD
lxc file push build/fcsc-agent dev-vm-1/tmp/fcsc-agent
lxc file push build/fcsc-agent.service dev-vm-1/tmp/fcsc-agent.service
lxc exec dev-vm-1 -- /tmp/install-agent.sh

# All VMs (edit deploy/vms.conf first)
./deploy/rollout.sh

# Agent only, specific VMs
./deploy/rollout.sh --agent-only dev-vm-1 dev-vm-3
```

### Update monitoring stack
```bash
cd monitoring-stack
docker compose up -d
```

## Data Flow

```
Bot (Rust)
  │
  ├── writes logs ──► metrics.log.YYYYMMDD.HHMMSS
  │                      │
  │                      ├──► fcsc-agent (tails, parses QUEUE lines)
  │                      │       │
  │                      │       └──► Prometheus scrapes :9100/metrics
  │                      │               │
  │                      │               └──► Grafana dashboards
  │                      │
  │                      └──► Promtail (ships to Loki)
  │                              │
  │                              └──► Loki ──► Grafana log explorer
  │
  ├── connects to ──► Geyser gRPC :10000 (Fastcar 1)
  │                      (health inferred from QUEUE log lines)
  │
  ├── connects to ──► ClickHouse :8123
  │                      (checked via SELECT 1 by fcsc-agent)
  │
  └── connects to ──► Solana RPC :8899 (Fastcar 1)
                         (checked via HTTP GET by fcsc-agent)
```

## Key Design Decisions

1. **No direct Geyser monitoring.** The bot already subscribes to Geyser and logs QUEUE statistics every ~10s. The agent reads these logs instead of making its own gRPC connection, avoiding extra load on Fastcar 1.

2. **CLI flags as source of truth.** Configuration lives in the systemd ExecStart line, making it visible via `systemctl cat fcsc-agent` and auditable without hunting for config files.

3. **Raw metrics only.** The agent exposes parsed values. State computation (green/yellow/red) happens in Grafana. This keeps the agent simple and lets the team iterate on thresholds without redeploying.

4. **This repo owns everything.** The monitoring stack configs, Grafana dashboards, agent source, deploy scripts, and promtail configs are all version-controlled here. Nothing on monitoring.lxd is hand-edited.

5. **Idempotent deployment.** Install scripts detect existing versions and skip unchanged files. Safe to re-run at any time.
